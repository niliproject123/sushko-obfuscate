- Apply only to this browser/device
- Persist across sessions
- Take priority over server defaults when processing

## User Workflow

### Typical Flow
1. Open app â†’ lands on User Config tab
2. Add custom replacements (e.g., specific names to replace)
3. Drag & drop PDF files
4. Click "Process PDFs"
5. Review results (matches found per page)
6. Download processed files

### Admin Flow
1. Switch to Admin Config tab
2. Modify detection patterns (enable/disable, edit regex)
3. Update replacement pools (add names, cities)
4. Adjust OCR settings if needed
5. Save changes (persists to server)

## Persistence

| Data | Storage | Lifetime |
|------|---------|----------|
| User replacements | localStorage | Until browser data cleared |
| Disabled detectors | localStorage | Until browser data cleared |
| Force OCR setting | localStorage | Until browser data cleared |
| Admin config | Server (settings.json) | Permanent |
| Processed files | Server temp storage | ~1 hour (TTL) |

| Config save failed | Error message in respective section |
